Session 7 · Iteration and debugging

Describe it, then change one layer

The traveling question: when something you care about breaks, what counts as a precise description — and which layer do you actually control?
The bridge from class

In this week's Instructor Study page, a spaceship and a block annihilate each other at generation 43 — and "the page is broken" bounces off the describe-gate; only a description a stranger could act on unlocks the tools. The fix panel offers three layers: the seed, the rule, and a brush that paints the outcome onto glass the rule never reads. One rule swap even turns the check green for the wrong reason.

That's the loop this page sends into your territory: observe, describe precisely, change one layer you control, rerun. Everything that breaks can be debugged like this — and some fixes are just paint.
